# LunaClock — project conventions

> Single-file static HTML app. Read `clock7.html` before editing — all UI, styles, and logic live in one file.

## Architecture

- **Ship file:** `clock7.html` — do not split into modules unless explicitly starting v8 with a new file
- **No npm / no bundler** — no `package.json`; CDN scripts only
- **Legacy:** `clock.html`–`clock6.html` are historical; do not patch unless backporting intentionally

## UX rules (non-negotiable)

- **Mobile-first** — test narrow viewport; no horizontal scroll
- **Pill buttons only** — rounded-full controls; no sharp rectangular chrome for actions
- **No Luna Pink preset** — monotone default palette; accent via inversion mode, not brand pink theme
- **Mode bar order:** Stopwatch → Clock → Pomodoro (Clock is default on load)
- **Mode indicator** matches active menu label (e.g. "Clock", not "Flip Clock")
- **Profile name** appears first in settings

## Inversion mode

- CSS variables in `:root` drive light theme; `.inverted` (or equivalent) flips to dark/high-contrast
- Preserve readability for teen UX — avoid harsh pure black/white without dimmed surfaces

## Settings & persistence

- Cookies + LUNA7 settings code + QR export/import
- Cookie path is pathname-derived — test with static server under deploy-like paths when debugging persistence

## External APIs / assets

- Mixkit sounds, Quotable API, QRCode.js CDN — handle offline/failure gracefully (no hard crash)
- Confetti on pomodoro/session complete — keep lightweight

## Editing discipline

- Minimize diff scope — one concern per session when possible
- Search within `clock7.html` before adding duplicate helpers
- No drive-by refactors across legacy `clock*.html` files

## Deploy

- **clock.petralian.com** — upload `clock7.html`; no build
- Manual smoke test checklist in `README.md`
